# Records FHIR Validation Skill Eval Results v0.2.1

## Scope

This release tightens the skill workflow after reviewing current AI-assisted FHIR validation patterns:

- Adds an explicit project-assessment phase before repository or IG validation.
- Protects generated FSH artifacts by tracing issues back to source files when available.
- Adds guardrails for complex slicing, terminology, profile inheritance, and reference-resolution fixes.
- Adds a Records-specific quality-rule derivation workflow.
- Extends regression evals for FSH source mapping and quality-rule derivation.

- `npm run test:claude-plugin`
- `npm run export:claude-plugin`
- `npx --yes @anthropic-ai/claude-code plugin validate dist/claude-records`

## Release Decision

Ship v0.2.1 as a prompt/workflow hardening release. No plugin API or installation changes.
